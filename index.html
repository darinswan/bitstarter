There are several DW systems on the market today but many have drawbacks and limitations. They aren't really designed for optimal performance and don't cache data where they should resulting in data thrashing, increased network traffic and more overhead than is needed to run the query. To make matters worse, the next query in the pipe does the same thing and in some cases the query pipeline is blocking. Moxy is a different approach that tries to run queries as fast as possible using various architectural approaches. It's no longer a one approach system for your data like many existing systems try to use.

Existing Data Warehouse Systems such as Greenplum and Vertica, try to solve this problem by using a Shared Nothing Architecture and Massively Parallel Processing (MPP). However, it really isn't a shared nothing architecture. And because of this approach some of the data has to be moved and analyzed over and over again depending on the system. The design is then left to the DW Architect to handle the various cases that come up and their options are limited. Make one query fast at the cost of another. Should we index? How do we shard the data? In most cases this depends on the data, which the architect doesn't always know, and doesn't know what other queries may be run later that will decrease performance over the entire system. The Designer has to make a best guess and just hope for the best, in a system where data can be of enormouse size, and take weeks to copy let alone adequately analyze. Additionaly, there are use cases where there just is no optimal solution because that's just how that particular Data Warehouse System does things. There needs to be a more dynamic approach to handling processing and storing the data.

This is where Moxy comes in. It's a different approach to Data Warehouse systems and allows the developer much higher levels of control over how and where data is stored and queried. This reduces many of the delays which also results in less hardware that can perform the same tasks. Faster queries, faster load and backup times, and minimumizes data re-distribution and network traffic.
